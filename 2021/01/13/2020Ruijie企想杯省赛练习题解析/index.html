3、网络设备安全技术

  • 为交换机和无线控制器开启SSH服务端功能，用户名和密码为admin，密码为明文类型,特权密码为admin。

Ruijie(config)#enable service ssh-server

Ruijie(config)#username admin PASsword admin

Ruijie(config)#enable password admin

  • 为路由器和出口网关开启Telnet功能，对所有Telnet用户采用本地认证的方式。创建本地用户，设定用户名和密码为admin，密码为明文类型,特权密码为admin。

Ruijie(config)#enable service telnet-server

Ruijie(config)#username admin PASsword admin

Ruijie(config)#enable password admin

  • 配置所有设备SNMP消息，向主机172.16.0.254发送Trap消息版本采用V2C，读写的Community为“ruijie”，只读的Community为“public”，开启Trap消息。

Ruijie(config)#snmp-server enable traps

Ruijie(config)#snmp-server community public rw

Ruijie(config)snmp-server community ruijie rw

Ruijie(config)snmp-server community public ro

​

Ruijie(config)# snmp-server host 172.16.0.254 traps version 2c ruijie

Ruijie(config)#snmp-server host 172.16.0.254 traps version 2c public

  1. 局域网环路规避方案部署

为规避网络末端接入设备上出现环路影响全网，要求在本部接入设备S1，S2进行防环处理。具体要求如下：

  • 终端接口开启BPDU防护不能接收 BPDU Guard报文；

Ruijie(config)#interface gigabitEthernet 0/1

Ruijie(config-if)#spanning-tree bpduguard enable

  • 终端接口下开启RLDP防止环路，检测到环路后处理方式为Shutdown-Port；

Ruijie(config)#rldp enable

Ruijie(config)#interface gigabitEthernet 0/1

rldp port loop-detect shutdown-port

  • 连接终端的所有端口配置为边缘端口；

int gi0/1

Spanning-tree portfast

  • 如果端口被 BPDU Guard检测进入 Err-Disabled状态，再过 300 秒后会自动恢复（基于接口部署策略），重新检测是否有环路；

Ruijie(config)#interface gigabitEthernet 0/1

Ruijie(config-if-range)#errdisable recovery interval 300

  • 规避高流量报文风暴对网络的冲击，在交换机S5上针对用户终端接口的广播，组播，未知名单播启用风暴限制，限制级别2；

Ruijie(config-if-GigabitEthernet 0/1)#storm-control multicast level 2

Ruijie(config-if-GigabitEthernet 0/1)#storm-control broadcast level 2

Ruijie(config-if-GigabitEthernet 0/1)#storm-control unicast level 2

  1. DHCP中继与服务安全部署

在交换机S3、S4上配置DHCP中继，对VLAN10内的用户进行中继，使得本部PC1用户使用DHCP Relay方式获取IP地址。具体要求如下：

  • DHCP服务器搭建于EG1上，地址池命名为Pool_VLAN10，DHCP对外服务使用loopback 0地址；

Ruijie(config)#int loopback 0

Ruijie(config-if-Loopback 0)#ip address 11.1.0.11 255.255.255.255

Ruijie(config)#int loopback 1

Ruijie(config-if-Loopback 0)#ip address 12.1.0.1 255.255.255.255

ip dhcp pool Pool_VLAN10

option 138 ip 11.1.0.11

network 192.168.10.0 255.255.255.0

default-router 192.168.10.252

为了防御动态环境局域网伪DHCP服务欺骗，在S1、S2交换机部署DHCP Snooping功能；

Ruijie(config)#ip dhcp snooping

Ruijie(config)#int rang gigabitEthernet 0/23 -24

Ruijie(config-if-GigabitEthernet 0/24)#ip dh snooping suppression

  • 为了防止大量网关发送的正常的相关报文被接入交换机误认为是攻击被丢弃，从而导致下联用户无法获取网关的ARP信息而无法上网，要求关闭S1/S2上联口的NFPP功能；

Ruijie(config)#int ra gigabitEthernet 0/23-24

Ruijie(config-if-range)#no nfpp arp-guard enable

  • 全局设置NFPP日志缓存容量为1024，打印相同log的阈值为300s;

Ruijie(config)#nfpp

Ruijie(config-nfpp)#log-buffer entries 1024

Ruijie(config-nfpp)#log-buffer logs 1024 interval 300

  • 调整CPU保护机制中ARP阈值500pps；

Ruijie(config)#cpu-protect type arp bandwidth 500

  • 为了防止伪 IP 源地址攻击， 导致出口路由器会话占满，要求S5交换机部署端口安全，接口Gi0/1只允许PC3通过。

Ruijie(config)#int gigabitEthernet 0/1

Ruijie(config-if-GigabitEthernet 0/1)#switchport mode access

Ruijie(config-if-GigabitEthernet 0/1)#switchport port-security

Ruijie(config-if-GigabitEthernet 0/1)#switchport port-security mac-address PC3mac

  1. MSTP及VRRP部署

在本部交换机S1、S2、S3、S4上配置MSTP防止二层环路；要求所有有线数据流经过S3转发，S3失效时经过S4转发。所配置的参数要求如下：

Ruijie(config)#spanning-tree

Ruijie(config)#spanning-tree mst configuration

  • region-name为ruijie；

Ruijie(config-mst)#name ruijie

  • revision版本为1；

Ruijie(config-mst)#revision 1

  • S3作为实例中的主根， S4作为实例中的从根；
  • 主根优先级为4096，从根优先级为8192；

S3(config)#spanning-tree mst 0 priority 4096

S4(config)#spanning-tree mst 0 priority 8192

  1. 全局流表策略部署

在用户没有防火墙做限制的情况下，如果遇到大量的伪源IP攻击，或者是端口扫描时，会把设备的流表给占满，而导致正常的数据无法建流而被丢弃，为此要求总部部署全局流表防火墙，ACL（编号为102）策略要求如下：

  • 放通所有IP到本设备外网接口的ICMP、Telnet协议;
  • 放通内网终端IP到外网所有资源的访问;
  • 放通任意IP来访问映射的内网交换机的资源;
  • 根据上下文要求放通设备已启用的功能协议端口。

Ruijie(config)#access-list 102 permit 23 any any

Ruijie(config)#access-list 102 permit ICMp ANy any

Ruijie(config)#access-list 102 permit ip any any

Ruijie(config)#int range gigabitEthernet 0/0-1

Ruijie(config-if-range)#ip access-group 102 in l

  • 出口网关上进行NAT配置实现本部与分校的所有用户均可访问互联网，通过NAPT方式将内网用户IP地址转换到互联网接口上，同时总部用户仅可在周一到周五工作时间09：00-17:00（命名为work）访问互联网；

Ruijie(config-)#interface GigabitEthernet 0/0

Ruijie(config-if-GigabitEthernet 0/0)#ip address 10.1.0.1 255.255.255.252

Ruijie(config-)#interface GigabitEthernet 0/2

Ruijie(config-if-GigabitEthernet 0/2)#ip add 20.1.0.6 255.255.255.248

Ruijie(config)#interface GigabitEthernet 0/3

Ruijie(config-if-GigabitEthernet 0/3)#ip address 30.1.0.6 255.255.255.248

Ruijie(config)#interface GigabitEthernet 0/4

Ruijie(config-if-GigabitEthernet 0/4)#ip address 40.1.0.6 255.255.255.248

Ruijie(config)#interface GigabitEthernet 0/0

Ruijie(config-if-GigabitEthernet 0/0)#ip nat inside

Ruijie(config-)#interface GigabitEthernet 0/2

Ruijie(config-if-GigabitEthernet 0/3)#ip nat outside

Ruijie(config)#interface GigabitEthernet 0/3

Ruijie(config-if-GigabitEthernet 0/3)#ip nat outside

Ruijie(config)#interface GigabitEthernet 0/4

Ruijie(config-if-GigabitEthernet 0/4)#ip nat outside

Ruijie(config)#ip nat pool work 50.1.0.1 50.1.0.1 netmask 255.255.255.252

Ruijie(config)#ip nat inside source list 101 pool work overload

Ruijie(config)#time-range work

Ruijie(config-time-range)#periodic weekdays 09:00 to 17:00

Ruijie(config)#access-list 101 permit ip any any time-range work

Ruijie(config)#access-list 101 deny ip any any

Ruijie(config)#int range gigabitEthernet 0/2-4

Ruijie(config-if-range)#ip access-group 101 out

  • 在本部EG1上配置，使本部核心交换S4（11.1.0.34）设备的SSH服务可以通过互联网被访问，将其地址映射至联通线路上，映射地址为20.1.0.2；

Ruijie(config)#int loopback 0

Ruijie(config-if-Loopback 0)#ip add 11.10.0.34 255.255.255.255

Ruijie(config)#int gigabitEthernet 0/2

Ruijie(config-if-GigabitEthernet 0/2)#ip add 20.1.0.6 255.255.255.252

Ruijie(config-if-GigabitEthernet 0/2)#ip nat outside

Ruijie(config)#int gigabitEthernet 0/24

Ruijie(config-if-GigabitEthernet 0/24)# ip nat inside

Ruijie(config)#ip nat inside source static 11.1.0.34 20.1.0.2 permit-inside

本部内网主机有访问上海办事处S6设备 的Telnet服务需求，但本部内网因网络规划要求不能引入外部路由，同时上海办事处网络运维人员考虑安全起见也不希望将S6设备（11.1.0.6）地址对外公布。为此规划在出口网关上进行NAT地址转化将S6真实地址映射至20.1.0.20。

Ruijie(config)#int loopback 0

Ruijie(config-if-Loopback 0)#ip add 11.1.0.6 255.255.255.255

Ruijie(config)#int gigabitEthernet 0/2

Ruijie(config-if-GigabitEthernet 0/2)#ip add 20.1.0.6 255.255.255.252

Ruijie(config-if-GigabitEthernet 0/2)#ip nat outside

Ruijie(config)#int gigabitEthernet 0/24

Ruijie(config-if-GigabitEthernet 0/24)# ip nat inside

Ruijie(config)#ip nat inside source static 11.1.0.34 20.1.0.2 permit-inside

  1. 全局流表策略部署

在用户没有防火墙做限制的情况下，如果遇到大量的伪源IP攻击，或者是端口扫描时，会把设备的流表给占满，而导致正常的数据无法建流而被丢弃，为此要求总部部署全局流表防火墙，ACL（编号为102）策略要求如下：

  • 放通所有IP到本设备外网接口的ICMP、Telnet协议;
  • 放通内网终端IP到外网所有资源的访问;
  • 放通任意IP来访问映射的内网交换机的资源;
  • 根据上下文要求放通设备已启用的功能协议端口。

Ruijie(config)#access-list 102 permit 23 any any

Ruijie(config)#access-list 102 permit ICMp ANy any

Ruijie(config)#access-list 102 permit ip any any

Ruijie(config)#int range gigabitEthernet 0/0-1

Ruijie(config-if-range)#ip access-group 102 in l

  1. Web Portal用户认证部署
  • 在本部网关上启用Web Portal认证服务，并创建user1、user2，密码均为XX（现场提供）；

Ruijie(config)#web-auth enable

Ruijie(config)#username user1 password admin

Ruijie(config)#username user2 password admin

  • 本部有线用户需进行WEB认证访问互联网；

  • 本部无线用户不需在EG上进行WEB认证即可访问互联网。

Ruijie(config)#web-auth direct-host 192.0.100.1

Ruijie(config)#web-auth direct-host 192.0.50.0

Ruijie(config)#web-auth direct-host 192.0.60.0
